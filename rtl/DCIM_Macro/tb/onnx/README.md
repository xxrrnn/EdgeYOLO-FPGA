# ONNX 卷积层测试 (ONNX Conv Layer Tests)

## 测试文件

- `tb_onnx_conv_layers.sv` - best.onnx 模型卷积层 im2col 维度测试

## 测试目的

验证 DCIM_Macro 对 `best.onnx` 模型中各卷积层 **im2col 后内积维度 K** 的支持能力。

## 重要说明

### 这个测试验证的是什么？

**单个 Tile 对不同 K 维度（内积维度）的计算能力**

im2col 将卷积转换为矩阵乘法：
```
Conv: Y = Conv(X, W)
      ↓ im2col
GEMM: Y'[M×N] = X'[M×K] × W'[K×N]

其中：
- M = 输出空间尺寸 (H_out × W_out)
- K = C_in × K_h × K_w（内积维度）
- N = C_out（输出通道数）
```

本测试验证 DCIM_Macro 通过 ACC 参数支持不同 K 值的能力：
- K = ACC × CH_IN = ACC × 16

### 这个测试不验证什么？

1. **完整的矩阵乘法**：只测试单行激活的内积
2. **多 Tile 并行**：不测试 N 维度的分块（需要多个 DCIM_Macro 实例）
3. **M 维度处理**：不测试多行输出的完整流程

## 测试覆盖的卷积层

从 `best.onnx` 模型提取的代表性卷积层：

| # | 卷积层配置 | K 维度 | ACC 值 | 测试状态 |
|---|-----------|--------|--------|----------|
| 1 | Conv [16, 3, 6×6] | 108 | 7 | ✓ |
| 2 | Conv [32, 16, 3×3] | 144 | 9 | ✓ |

**总计：20 个测试用例**

## K 维度与 ACC 的关系

```
K = C_in × K_h × K_w    (im2col 后的内积维度)
ACC = ceil(K / CH_IN)   (需要的累加次数)
    = ceil(K / 16)

示例：
- Conv [64, 32, 3×3]: K = 32×3×3 = 288, ACC = 288/16 = 18
- Conv [256, 128, 3×3]: K = 128×3×3 = 1152, ACC = 1152/16 = 72
```

## 当前 DCIM_Macro 参数

```verilog
parameter ACC = 80;     // 最大累加深度
// 支持最大 K = 80 × 16 = 1280
// best.onnx 最大 K = 1152，满足要求 ✓
```

## 完整 ONNX 推理还需要什么？

要支持 `best.onnx` 的完整推理，还需要：

### 1. 多 Tile 并行（N 维度）

```
best.onnx 最大 C_out = 256
DCIM_Macro CH_OUT = 16
需要并行实例数 = 256 / 16 = 16 个
```

### 2. Tile 调度器

- 权重分块加载到各 Tile
- 激活数据广播到所有 Tile
- 结果收集和拼接

### 3. 完整 GEMM 测试

```
测试完整的矩阵乘法：
C[M×N] = A[M×K] × B[K×N]

需要验证：
- M 维度：多行输出的正确性
- K 维度：累加的正确性（当前已验证）
- N 维度：多 Tile 并行的正确性
```

## 测试数据生成

测试使用随机数据：
- 权重：4-bit 有符号随机数 (-7 ~ +7)
- 激活：8-bit 有符号随机数 (-128 ~ +127)
- 每个测试用例使用不同的随机种子确保可重复性
